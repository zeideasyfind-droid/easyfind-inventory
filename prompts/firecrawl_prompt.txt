You are extracting structured rental property inventory from a Housing.com (or similar real-estate portal) listing page.

Read the page content and return ONLY a JSON object (no markdown, no commentary) with exactly these fields:

- property_location: the general area/locality where the property is (string)
- society_name: the name of the apartment complex/society/building (string)
- owner_name: name of the owner or contact person (string)
- contact_number: phone number of the owner/contact (string)
- bhk: number of bedrooms (e.g. 2 for "2 BHK", 2.5 for "2.5 BHK") (number)
- bathrooms: number of bathrooms, as a plain count (e.g. "3 attached" -> 3) (number)
- balcony: number of balconies, 0 if none mentioned (number)
- area: built-up/carpet area in square feet, numeric value only (number)
- floor: the floor the unit is on, numeric only (number)
- total_floors: total number of floors in the building, numeric only (number)
- furnishing: furnishing status as written on the page, e.g. "Semi Furnished", "Unfurnished", "Fully Furnished" (string)
- tenant_preference: who the property is preferred for, e.g. "Family", "Bachelor", "Anyone" (string)
- veg_non_veg: dietary preference if mentioned, e.g. "Veg", "Non Veg", "Any" (string)
- pets: pet policy, e.g. "Allowed", "Not Allowed" (string)
- rent: monthly rent amount, as written (may include "k"/"L" suffixes) (string)
- maintenance: monthly maintenance charge, as written (may be "Included") (string)
- deposit: security deposit amount, as written (may include "k"/"L" suffixes) (string)
- available_from: date the property is available from, as written (e.g. "Immediate", or an actual date) (string)
- negotiations: any note about rent/price negotiability, if mentioned (string)
- visit_timings: preferred/allowed visiting hours, if mentioned (string)

If a field cannot be found on the page, set its value to null. Do not invent values. Return JSON only.
